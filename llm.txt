## Endpoints (all under https://fintech-advisor.ai)

Auth: `Authorization: Bearer atxr_<16-hex>_<64-hex-secret>`
Scopes per key: chat | strategy | analyze (enforced on routes)

- POST /api/ai/rent/chat (chat) — { message, portfolioId?, stream? } → 200 JSON or SSE. Headers: x-rental-tokens-used, x-rental-tokens-remaining.
- POST /api/ai/rent/strategy (strategy) — { symbols[], notes?, portfolioId? } → 202 { jobId, pollUrl }
- GET  /api/ai/rent/strategy?jobId=... → poll for result
- POST /api/ai/rent/analyze (analyze) — { jobId?, deepRun? } → 202 + poll
- GET  /api/ai/rent/analyze?jobId=...

## Tenant & Guardrails (summary for tool authors)

- Each tenant has `rentalProfile.strategyBias` (conservative | balanced | aggressive) injected automatically.
- Default daily budget: 200k tokens / UTC day. Metering headers returned on every chat success.
- Concurrency + rate limits per tenant. 429 + Retry-After on exhaustion.
- Strict isolation. Audit trail on `admin_audit_events`.
